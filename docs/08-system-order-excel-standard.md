# 系统运单明细 Excel 上传规范

## 适用文件

后续上传 Excel 默认按 `2026.6月系统运单明细.xlsx` 的结构处理。

文件可能包含：

1. 系统透视汇总 sheet：用于人工查看，不作为逐行财务分析源。
2. 系统明细 sheet：包含 `运单号、收付类型、费用类型` 等字段，是分析唯一明细源。

系统导入和 FP&A 分析必须自动寻找包含 `运单号、收付类型、费用类型` 的工作表，不得默认读取第一个 sheet。

## 明细字段

系统明细至少应包含：

`运单号、客户订单号、用户、服务、收费重(KG)、供应商收费重(KG)、供应商、供应商服务、收付类型、费用类型、金额、单价、本币费用、销售代表、备注、折合人民币、客服代表、下单时间、内部备注、实重、件数、主品名`

其中 `备注` 可能出现两列：一列保存原始汇率/币种说明，一列保存系统使用汇率。

## 金额口径

1. 不使用外部汇率 API。
2. 不优先使用系统 `折合人民币` 作为分析金额；分析金额按 `金额 × 原始表格标注汇率` 重新计算。
3. 原始表格标注 `1` 时按人民币计算。
4. 原始表格标注 `美金/美元/USD/$/美金汇率未出/汇率未出` 时统一按 6.85。
5. 原始表格标注其他数字汇率时，严格按该数字计算。
6. 应收为正数，应付可能为负数；分析时应付取绝对值进入应付合计。
7. 派送费中原始录入 700、800 等美金金额时，按原始金额 × 6.85，不得固定按 600 美金。
8. 清关费或派送费缺少应付时，默认利润 100 美金，即 685 元，并倒推应付成本。

## 分类口径

物流业务：汽运灰关、汽运白关拼车、铁路白关整柜等。

注册/服务类业务：注册、EAC 证书、商标注册、店铺租赁、公司注册/注销等，必须单独成表并由主管确认，不进入物流提成。

## 必出分析

每次月度分析至少输出：

1. CFO 管理层摘要
2. 月度营收毛利总览
3. 单票毛利明细
4. 业务类型汇总
5. 高风险订单清单
6. 异常高利润订单清单
7. 应收应付对账问题
8. 异常&成本补录清单
9. 上游应付分析
10. 业务员订单提成明细
11. 业务员提成汇总
12. 注册业务价格提成表
13. 操作员绩效
14. 客户利润分析
15. 原始数据
16. 参数规则
17. 数据校验表

## 校验要求

所有汇总金额必须能追溯到单票明细和原始行号。数据校验表至少检查：

1. 明细应收合计 = 总览应收金额。
2. 明细应付合计 = 总览应付金额。
3. 明细毛利合计 = 总览调整后毛利。
4. 业务类型利润合计 = 明细毛利合计。
5. 业务员提成明细合计 = 提成汇总合计。
6. 原始行数已全部处理。
7. 订单数已全部分类。
