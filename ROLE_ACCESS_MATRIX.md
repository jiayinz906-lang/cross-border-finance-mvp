# XJD Finance 角色、界面与权限矩阵

本文件是账号分工的产品说明。后端 `server/src/config/rbac.ts` 是权限判定的唯一权威来源，前端菜单和按钮只负责改善操作体验，不能替代后端校验。

## 数据范围

| 角色 | 数据范围 |
| --- | --- |
| 系统管理员 | 全部业务数据 |
| 财务 | 全部业务数据 |
| 主管 | 全部业务数据 |
| 老板/管理层 | 全部业务数据，只读 |
| 销售代表 | 仅 Excel 中“销售代表”与本人账号显示名称或账号一致的订单、提成和确认单 |
| 操作员 | 仅 Excel 中“客服代表/操作员”与本人账号显示名称或账号一致的订单、绩效和确认单 |
| 未授权 | 无业务数据 |

## 使用约束

1. 销售和操作员账号的显示名称应与每月 Excel 中对应人员姓名完全一致；用户名也可作为备用匹配值。
2. 页面不可见、按钮不可见和接口拒绝是三层控制。最终以接口返回的 401/403 为准。
3. 锁账、确认单、风险复核和收付款等关键动作继续写入审计日志。
4. 修改角色后，旧登录令牌在下一次接口校验时会读取数据库中的最新角色；用户重新登录可立即刷新菜单说明。
