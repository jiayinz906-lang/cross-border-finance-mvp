# XJD Finance 跨境物流财务管理使用手册

## 1. 访问地址

| 环境 | 地址 | 用途 |
|---|---|---|
| 本地前端 | `http://localhost:5173/` | 本机测试和日常操作 |
| 本地后端 | `http://localhost:4000/api` | API 健康检查与本地开发 |
| 线上前端 | `https://jiayinz906-lang.github.io/cross-border-finance-mvp/#/dashboard` | 外部用户访问系统 |
| 线上后端 | `https://cross-border-finance-server.onrender.com/api` | 线上 API |

> 本地地址需要电脑、前端服务、后端服务和本地 PostgreSQL 均处于运行状态。线上地址不依赖本机开机。

## 2. 初始账号与密码

下列为系统初始化时创建的默认账号。若任一账号已在系统中修改密码、被管理员重置或被停用，应以参数规则页的账号管理记录为准。

| 角色 | 账号 | 默认密码 | 主要权限 |
|---|---|---|---|
| 系统管理员 | `admin` | `admin123` | 用户、导入、规则、确认、锁账、日志和通知配置 |
| 财务 | `finance` | `finance123` | 导入、财务核对、应收应付、风险与报表 |
| 主管 | `supervisor` | `supervisor123` | 注册提成确认、风险复核、主管确认、锁账 |
| 老板/管理层 | `boss` | `boss123` | 经营总览和管理层查看 |
| 销售/客服 | `sales` | `sales123` | 查看本人相关业务及确认单 |

**首次登录后立即修改密码。** 管理员在“参数规则 -> 账号与权限”可创建用户、停用账号、重置密码，以及维护员工钉钉用户 ID。

## 3. 日常月度操作流程

1. 使用财务或管理员账号登录。
2. 在“经营总览”右上角选择要处理的月份。
3. 点击“上传 Excel 导入”，选择符合表头模板的原始台账。
4. 在导入预览中核对文件名、月份、工作表、表头映射、应收、应付、毛利、风险和阻断项。
5. 有阻断项时，按提示修改 Excel 的对应行和字段后重新预览；无阻断项后点击“确认写入数据库”。
6. 导入成功后，系统自动切换到导入月份；依次处理风险、注册提成、物流提成、操作员绩效、应收应付与薪资确认。
7. 所有事项完成后，由主管在“参数规则 -> 月度闭环进度”执行锁账。

## 4. Excel 导入规则

### 4.1 两步导入

- **预览阶段不写入数据库**：只检查表头、金额、汇率、订单和质量问题。
- **确认写入才入库**：生成导入批次、原始 Excel 行和费用明细账。
- 同月份重新导入时，系统以最新有效批次作为统计来源；锁账月份不能覆盖导入。

### 4.2 数据口径

- 所有应收、应付、毛利均由导入 Excel 的费用明细行汇总，保留原始正负号。
- 人民币按原表金额；美元按原表明确汇率，没有明确汇率时按系统规则处理。不得手工改写原始费用金额。
- 赔付、改单费、续页费、保险费等属于费用明细，按原始方向和金额参与订单及供应商汇总。
- 原始数据可从“风险复查 -> 原始数据”或应收/应付订单明细下钻，查看 Excel 行号、费用类型、金额、汇率、供应商、销售代表、客服代表和用户。

## 5. 页面说明

### 经营总览

- 查看当月总应收、总应付、调整后毛利、毛利率、总票数、物流提成和高风险票数。
- 查看业务利润、业务员毛利排行、客户利润和上游应付集中度。
- 月份切换后所有页面统一使用该月份的最新有效导入批次。

### 业务利润

- 同时展示总业务、物流业务和注册/服务业务。
- 每个业务类型的应收、应付、毛利和毛利率均可追溯至订单与费用行。

### 物流提成

1. 按销售代表查看物流订单和汇总毛利。
2. 点击票数查看订单明细，可查看系统单号、原始订单号、毛利、提成比例和提成金额。
3. 调整单票提成比例时必须保存，已主管确认的历史单据不能直接覆盖。
4. 点击“生成个人确认单”，再从电子签名确认页外发或下载。

### 注册提成

- 显示导入 Excel 中的用户、销售代表、客服代表、服务、成交单价和成交利润。
- 主管填写或确认提成金额后保存。注册、证书、商标、店铺租赁等服务提成会归属对应销售代表，并进入薪资确认单。

### 电子签名确认

- 汇总销售代表提成薪资确认单和操作员薪资确认单。
- “查看确认单”显示确认单快照；Excel、PDF、PNG 均从同一份快照导出，金额应一致。
- “生成外发链接/发送钉钉”生成个人公开签名链接。员工打开链接核对明细后签名；签名成功后 token 失效。
- 主管确认、作废重签均保留版本和操作记录。已主管确认的确认单只能作废后生成新版本。

### 操作员绩效

- 以导入 Excel 中的客服代表/操作员为人员归属。
- 实际票数从 Excel 统计；规则基础票数按规则自动匹配。
- 可修改绩效单价、发放说明和允许调整的绩效字段，系统自动汇总各分类绩效金额和总绩效金额。
- 空运白关按固定 `50 元/票` 计算；其他分类按当前规则卡片和表格说明执行。

### 客户利润分析

- 按客户查看应收、应付、毛利和毛利率。
- 支持按应收规模或毛利规模比较客户，并下钻核对订单。

### 风险复查

- 低毛利、异常高利润、汇率缺失、负毛利、缺应付等订单进入风险列表。
- 点击“原始数据”核对导入的 Excel 行；填写复核说明并保存复核结果。

### 应收管理

- 查看客户应收、账龄和订单明细。
- 可登记回款、查看回款记录和作废错误回款；操作后账龄及未收款金额会刷新。

### 上游应付

- 按供应商汇总应付、已付款、未付款、账龄和订单数。
- 可下钻到订单费用明细，登记付款或作废错误付款。

### 参数规则

- 管理业务规则、汇率口径、导入模板、月度闭环、账号权限和操作日志。
- 配置钉钉企业应用或群机器人凭据，并为员工维护钉钉用户 ID。
- 月度闭环显示导入审计、风险复核、注册确认、物流提成签名、操作员签名和应收应付对账状态；所有阻断项解决后才可锁账。

## 6. 个人电子签名操作

1. 管理员或主管在“电子签名确认”中点击“生成外发链接”或“发送钉钉”。
2. 员工收到链接后打开网页，核对姓名、月份、订单/绩效明细、毛利、提成比例和确认金额。
3. 在签名页提交电子签名。
4. 系统记录签名时间、IP、浏览器信息，并将链接设置为不可重复使用。
5. 主管在后台核对后点击“主管确认”；如需更正，先作废重签并说明原因。

## 7. 钉钉通知使用前检查

1. Render 后端环境变量已配置 `DINGTALK_APP_KEY`、`DINGTALK_APP_SECRET` 和 `DINGTALK_ROBOT_CODE`，或群机器人 Webhook 配置。
2. 钉钉企业内部应用已获得向个人或群发送消息权限并完成发布。
3. 参数规则页中，员工账号已维护正确的钉钉用户 ID。
4. 先用一名测试员工生成确认单并发送；若自动发送失败，页面会显示完整链接，可手工复制发送。

## 8. 常见问题

| 问题 | 处理方式 |
|---|---|
| 登录失败 | 确认账号、密码；默认密码仅适用于未修改过的初始化账号。联系管理员重置密码。 |
| 本地网页打不开 | 检查前端 `5173`、后端 `4000` 和本地 PostgreSQL `54329` 是否运行。 |
| 导入按钮不可用 | 检查是否已登录、Excel 是否有阻断项、目标月份是否已锁账。 |
| 金额与 Excel 不一致 | 在风险复查或订单费用明细中按 Excel 行号核对；不要直接修改汇总金额。 |
| 下载提示 401/无权限 | 重新登录后再下载，下载操作需要有效登录 token。 |
| 钉钉发送失败 | 检查 Render 环境变量、钉钉应用权限/发布状态和员工钉钉用户 ID。 |

## 9. 管理要求

- 原始 Excel 是金额来源，确认单与下载文件使用生成时的快照，不重新计算历史金额。
- 导入前先预览，导入后先核对风险和应收应付，再执行提成与签名。
- 锁账后禁止覆盖导入、回滚批次、重生成确认单和修改参数；如确需处理，由主管解锁并保留操作日志。
- 生产环境不要继续使用默认密码。管理员应为每名员工创建独立账号并启用钉钉用户映射。
